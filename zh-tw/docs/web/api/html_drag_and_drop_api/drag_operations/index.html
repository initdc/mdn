Updates to DataTransfer.types

Note that the latest spec now dictates that DataTransfer.types (en-US) should return a frozen array of DOMStrings rather than a DOMStringList (en-US) (this is supported in Firefox 52 and above).

As a result, the contains (en-US) method no longer works on the property; the includes method should be used instead to check if a specific type of data is provided, using code like the following:

if ([...event.dataTransfer.types].includes('text/html')) {
  // Do something
}

You could always use some feature detection to determine which method is supported on types, and run code as appropriate.
